# EXPAND-001: 全国銭湯組合サイト スクレイパー拡張

## なぜ必要か

現在のバッチは東京（1010.or.jp、約410件）専用。全浴連（全国公衆浴場業生活衛生同業組合連合会）傘下の47都道府県組合がそれぞれ公式サイトを持ち、各サイトに公式データ（名前・住所・営業時間・定休日・電話番号）が掲載されている。これが最も精度の高い一次データ源。

## 選択理由

| データ源 | 精度 | コスト | 備考 |
|---|---|---|---|
| 各都道府県組合サイト | ◎ | 無料 | 公式データ、営業時間・料金まで含む |
| OSM Overpass API | ○ | 無料 | 座標補完用途に限定（都市部◎、地方△） |
| Google Places API | ◎ | 有料 | データ二次配布禁止のため除外 |
| Yahoo! ローカルサーチAPI | ○ | 無料 | 件数が組合サイトより少ない可能性 |

## やること

### フェーズ1: 全国組合サイト調査（都道府県ごと）

- [ ] 全浴連の都道府県組合一覧（`https://www.zenyoku.1010.or.jp/union/`）を取得
- [ ] 各都道府県サイトの以下を確認：
  - robots.txt の制限（既存調査では管理画面のみ Disallow でコンテンツページは可）
  - HTML 構造（静的 SSR か JS 動的ロードか）
  - 取得可能フィールド（緯度経度の有無が重要）
  - ページネーションの有無と構造
- [ ] 優先度付け：件数が多く構造が類似したサイトをまとめて実装
  - 優先: 大阪（osaka268.com）、京都（1010.kyoto）、愛知（aichi1010.jp）、福岡（fukuoka1010.com）、神奈川・埼玉・千葉

### フェーズ2: スクレイパー設計変更

現在の `batch/scraper.py` は東京専用のハードコードになっている。以下に変更：

```
batch/

├── fetcher.py        # 既存（変更なし）
├── parser.py         # 既存（東京用）
├── parsers/
│   ├── __init__.py
│   ├── base.py       # 共通インターフェース
│   ├── tokyo.py      # 既存ロジックを移植
│   ├── osaka.py      # 新規
│   ├── kyoto.py      # 新規
│   └── ...
└── db.py             # 既存（prefecture カラム追加に対応）
```

### フェーズ3: 主要都市の実装

優先順位（件数・観光利用頻度・構造の調査しやすさ）：

1. 大阪（osaka268.com） - 約268件
2. 京都（1010.kyoto） - 約84件
3. 神奈川（kanagawa1010.jp 等）
4. 愛知（aichi1010.jp） - 約49件
5. 福岡（fukuoka1010.com） - 約19件（掲載数少ない）

## 完了条件

- [ ] 上記5都市のスクレイパーが動作する
- [ ] `--region` または `--all` オプションで全国一括実行可能
- [ ] 各パーサーにドライランテストが通る（--dry-run --limit 3）

## 注意

- 各サイトのリクエスト間隔は最低2秒
- 利用規約の確認を各サイトで行うこと（東京と同様に記録）
- 緯度経度がないサイトは EXPAND-002（OSM 座標補完）と連携

## 依存

- EXPAND-002（DB スキーマ拡張）が先に完了している必要あり
- EXPAND-003（OSM 座標補完）と並行可能
