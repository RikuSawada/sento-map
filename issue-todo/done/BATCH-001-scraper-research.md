# BATCH-001: スクレイピング事前調査

## 概要
`batch/scraper.py` の実装前に、1010.or.jp の robots.txt・利用規約を確認し、
スクレイピングの法的・技術的可否を判断する。実装の GO/NOGO をこのチケットで決定する。

## やること
- [ ] `https://www.1010.or.jp/robots.txt` を確認し、禁止パスがないか確認
- [ ] `https://www.1010.or.jp/` の利用規約ページを確認し、データ利用制限を確認
- [ ] 地図ページ `https://www.1010.or.jp/map/` の HTML 構造を確認
  - 銭湯データが HTML 内に静的に含まれるか、JS で動的ロードされるか
  - 動的ロードの場合は API エンドポイントを特定する
- [ ] 取得可能なデータ項目を洗い出す（名前・住所・緯度経度・電話番号・URL・営業時間・定休日）
- [ ] 緯度経度が取得できない場合、住所からジオコーディングが必要かを判断
  - Google Geocoding API or 国土地理院 API の利用を検討
- [ ] 調査結果を `ideas/20260303-scraping-research.md` に記録

## 完了条件
- スクレイピング可否の判断が文書化されている
- 取得できるデータ項目・HTML 構造が明確になっている
- BATCH-002 の実装方針が決まっている

## 注意
- この調査は実際のリクエストを伴う。最低限のリクエストに留めること
- NOGO の場合は代替データソースを検討すること

## 依存
なし（最初に着手可能）
